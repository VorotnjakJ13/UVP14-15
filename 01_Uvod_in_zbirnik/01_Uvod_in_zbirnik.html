Raunalnik

  • CPE centralna procesna enota (mikroprocesor).
  • RAM zaasni spomin (Random Access Memory).
  • Vhodne enote tipkovnica, mika, …
  • Izhodne enote monitor, tiskalnik, …
  • Trajno shranjevanje diski, DVD, USB kljui, oblak, …

Delovanje raunalnika

  • CPE kontrolira vse po programu, ki je v pomnilniku RAM, in po e vgrajenih protokolih.
  • V spominu RAM so programi in podatki s katerimi trenutno dela CPE.
  • Vhodne enote piejo podatke na tono doloena mesta v RAM-u.
  • Izhodne enote berejo podatke iz tono doloenih mest v RAM-u.
  • Enote za trajno shranjevanje trajno hranijo podatke in programe.

Fizine komponente raunalnika

  • Mikroprocesor slui kot CPE.
  • Matina ploa slui kot povezovalnik.
  • RAM in CACHE spominski moduli sluijo kot zaasni pomnilnik.
  • Trdi diski (magnetni, SSD), DVD, USB kljui, mreni diski v oblaku (Dropbox, Google drive) sluijo kot enote trajnega spomina.
  • Zaslon (grafina kartica), USB vrata, zvoniki (zvona kartica), tipkovnica, mika, … sluijo kot vhodno-izhodne enote.

Kako deluje raunalnik?

  • Mikroprocesor vsebuje nekaj spominskih celic (registri), nad katerimi izvajamo aritmetino-logine operacije.
  • RAM (spomin) je skladie otevilenih spominskih celic. Vsaka celica hrani 1 Byte (zlog). tevilkam celic pravimo naslovi.
  • V delu RAM-a se nahaja zapis programa.
  • Mikroprocesor sledi zapisu programa in izvaja ukaze.
  • Ukazi so lahko:
    • nalaganje vsebine iz celic v registre ali obratno,
    • aritmetino logine operacije v registrih,
    • pogojne vejitve, skoki v programu (navadni, funkcijski).

Programiranje raunalnika

  • Raunalnik je "totalno neumen".
  • Natanno in hitro lahko izvaja preproste operacije.
  • Programiranje: manipuliranje z ukazi, da doseemo da bo mikroprocesor naredil to kar hoemo, pri emer se mikroprocesorju niti ne sanja, kaj v resnici dela.
  • POMEMBNO:
    • raunalnik ne ve isto ni - niti zase (nima zavesti);
    • s programiranjem ga zmanipuliramo, da naredi nekaj uporabnega, eprav ne ve kaj v resnici dela;
    • raunalnik sam po sebi ne premore nobene inteligence;
    • je pa zelo hiter.

Simulator 8-bitnega raunalnika

  • Prvi mikroprocesor leta 1971 je bil 4-bitni.
  • Prvi procesor v PC raunalnikih je bil 8-bitni (Intel 8086 oz. 8088).
  • Simulator: (http://schweigi.github.io/assembler-simulator/)
    • precej dobro zajame delovanje 8-bitnih procesorjev;
    • dananji procesorji so 64-bitni, a delujejo na zelo podoben nain.

Simulator 8-bitnega raunalnika

  • delovni registri A, B, C in D:
    • izvajanje in shranjevanje rezultatov aritmetino-loginih operacij (setevanje, primerjava, …),
    • branje iz pomnilnika in pisanje vanj.
  • register IP (instruction pointer):
    • kae na trenutni ukaz v pomnilniku (hrani tevilko celice);
    • po opravljenem ukazu se vrednost registra IP povea (na naslednji ukaz);
    • vrednost IP lahko tudi spremenimo - izvajanje programa nadaljujemo na drugem delu.

Simulator 8-bitnega raunalnika

  • register SP (stack pointer):
    • kae na sklad, to je del pomnilnika, na katerega zaporedoma nalagamo vrednosti;
    • vsaki, ko naloimo vrednost, se vrednost registra SP zmanja, ko odstranimo vrednost pa povea.
  • Zastavice Z (zero), C (carry) in F (fail):
    • hranijo en bit (logina vrednost);
    • odvisne so od rezultata zadnje izvedene operacije;
    • na podlagi njih se odloamo o skokih v programu.
  • 256 zlogov RAM.
  • Zadnjih 24 zlogov pripada izhodni enoti (simulacija zaslona).
  • Zlog tik pred tem (naslov 231) je dno sklada.

Ukazi zbirnika

  • Splona sintaksa:
  • neobvezna_oznaka:  IME_UKAZA argumenti  ; neobvezen komentar
    • Preko oznake se lahko skliemo na naslov pomnilnika, kjer je ukaz.
    • Za imenom ukaza se lahko nahajajo z vejicami loeni operandi. Teh je lahko 0, 1 ali 2.

Operand je lahko

  • Vsebina registrov A, B, C, D, SP (samo pri ukazu MOV).
  • Vsebina v delovnih registrih interpretirana kot naslov, npr. [A].
  • Naslov v registru z odmikom: [D-3].
  • Konstanta: 0 … 255.
  • Konstanta interpretirana kot naslov, npr. [100].
  • Oznaka (interpretirana kot naslov).

Primerjava: CMP

Ukaz CMP odteje od prvega argumenta vrednost drugega:

CMP reg1, reg2
CMP reg1, constant
  • e sta vrednosti enaki, je rezultat 0 in se nastavi zastavica Z.
  • e je prva vrednost strogo manja od druge, pride do "negativne" vrednosti oz. pojavi se prenos (zastavica C).
  • e je prva vrednost strogo veja od druge, se ne nastavi nobena od zastavic C oz. Z.

Funkcije

  • Funkcija je del kode, na katere izvajanje zaasno skoimo s klicem funkcije, kodo jo izvedemo in se po zakljuku vrnemo na del kode takoj po klicu funkcije.

  • Klic funkcije: CALL.

    • Na sklad shrani naslov naslednjega ukaza in IP nastavi na zaetek funkcije

      CALL address

  • Vraanje iz funkcije: RET

    • Iz sklada prebere vrednost v IP in nadaljuje izvajanje programa.

      RET

Zakljuek programa

Program zakljuimo z ukazom HLT.

Hello world!

; Simple example
; Writes Hello World to the output

          JMP start
hello:    DB "Hello World!"     ; Variable
          DB 0                  ; String terminator

start:    MOV C, hello          ; Point to var 
          MOV D, 232            ; Point to output
          CALL print
          HLT                   ; Stop execution

print:    PUSH A                ; print(C:*from, D:*to)
          PUSH B
          MOV B, 0
.loop:    MOV A, [C]            ; Get char from var
          MOV [D], A            ; Write to output
          INC C
          INC D  
          CMP B, [C]            ; Check if end
          JNZ .loop             ; jump if not
          POP B
          POP A
          RET

Minimalni element v tabeli

; Najmani element v zaporedju (tabeli)

          JMP start
stev_el:  DB 5     ; tevilo elementov v tabeli
tabela:   DB 7     ; Tabela
          DB 2
          DB 8
          DB 3
          DB 5
min:      DB 0     ; Na koncu bo tu min 

start:    MOV C, tabela       ; Naslov zaetka tabele 
          ADD C, [stev_el]    ; Naslov zadnjega elementa + 1
          MOV A, tabela       ; Kazalec na zaetek tabele
          MOV B, [tabela]     ; V B trenutni min
          INC A               ; Premik kazalca za en element
.zanka:   CMP A, C            ; Ali je kazalec A presegel tabelo?
          JZ konec            ; e je, konamo zanko
          CMP B, [A]          ; Ali smo nali manji element 
          JBE povec           ; e nismo, ...
          MOV B, [A]          ; sicer ga zapiemo v B
povec:    INC A               ; Premik kazaleca naprej
          JMP .zanka          ; Ponovimo zanko
    
konec:    MOV [min], B        ; Shranimo min
          HLT

Urejanje z izbiranjem

; Urejanje z izbiranjem

          JMP start
stev_el:  DB 5        ; tevilo elementov v tabeli
tabela:   DB 7        ; Tabela
          DB 2
          DB 8
          DB 3
          DB 5

; rabimo C za konec, A za start, B za min

start:    MOV A, tabela      ; Zaetek tabele
          MOV C, tabela
          ADD C, [stev_el]  ; Konec tabele + 1
          CALL uredi        ; Klic funkcije
          HLT

Urejanje z izbiranjem - najdi

; najdi: A - zaetek, C - konec + 1, B - pozicija najmanjega elementa

najdi:    PUSH A              ; Shranimo morebitne stare vrednosti
          PUSH D
          MOV B, A     
          INC A               ; Premaknemo A na 2. element
.zanka:   CMP A, C          ; Ali je kazalec v A e presegel tabelo
          JZ konec1         ; e je, konamo zanko
          MOV D, [B]
          CMP D, [A]    
          JBE povec         ; e ne (B <= [A]) si ga ne zapomnimo
          MOV B, A          ; Sicer pa ga zapiemo v B
povec:    INC A             ; Premaknemo kazalec za eno mesto naprej
          JMP .zanka        ; Ponovimo zanko
konec1:   POP D     ; Obnovimo stare vrednosti 
          POP A
          RET

Urejanje z izbiranjem - uredi

; Urejanje: A - zaetek tabele, C - konec + 1,   
; A bo trenutna pozicija v tabeli
uredi:    PUSH A        ; shranjevanje registrov
          PUSH B
          PUSH D
.zanka2:  CMP A, C  ; je A dosegel C?
          JZ konec2
          CALL najdi    ; najdi minimum
          MOV D, [A]    ; zamenjava minimuma z [B]
          PUSH C
          MOV C, [B]
          MOV [A], C
          MOV [B], D
          POP C
          INC A     ; pomik po tabeli naprej
          JMP .zanka2
konec2:   POP D     ; obnova registrov
          POP B
          POP A
          RET
